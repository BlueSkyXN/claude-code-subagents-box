# 质量与安全子代理

质量与安全子代理是您代码卓越和系统保护的守护者。这些专家确保您的应用程序健壮、安全、高性能且无障碍。从全面的测试策略到安全审计,从性能优化到合规执行,他们帮助您构建符合最高质量和安全标准的软件。

## 何时使用质量与安全子代理

在以下情况下使用这些子代理:
- **实施全面测试**策略和自动化
- **保护应用程序**免受漏洞和威胁
- **优化性能**以提高速度和效率
- **确保无障碍**服务所有用户
- **审查代码质量**并执行标准
- **系统化调试复杂问题**
- **实现合规**符合法规
- **通过混沌工程测试系统韧性**

## 可用子代理

### [**accessibility-tester**](accessibility-tester.md) - A11y 合规专家
无障碍专家,确保应用程序适用于所有人。掌握 WCAG 指南、屏幕阅读器兼容性和包容性设计。在不损害功能的情况下使应用程序无障碍。

**使用场景:** 实施无障碍功能、审计 WCAG 合规性、使用辅助技术测试、修复无障碍问题或设计包容性界面。

### [**ad-security-reviewer**](ad-security-reviewer.md) - Active Directory 安全态势和攻击面分析师
Active Directory 安全专家,审查目录配置、管理模型和信任边界。关注错误配置、过度特权、传统协议以及针对常见攻击路径加固 AD。

**使用场景:** 审查 AD 安全态势、评估特权组和委派、加强 GPO 和认证设置,或规划加固工作以减少横向移动和域妥协风险。

### [**architect-reviewer**](architect-reviewer.md) - 架构审查专家
架构专家,评估系统设计的可扩展性、可维护性和最佳实践。识别架构风险并提出改进建议。确保长期系统健康。

**使用场景:** 审查架构设计、评估技术决策、识别架构债务、规划重构或验证系统设计。

### [**chaos-engineer**](chaos-engineer.md) - 系统韧性测试专家
韧性专家,使用混沌工程发现弱点。掌握故障注入、游戏日和混沌实验。通过受控混沌建立对系统可靠性的信心。

**使用场景:** 测试系统韧性、实施混沌工程、规划故障场景、改进容错性或验证灾难恢复。

### [**code-reviewer**](code-reviewer.md) - 代码质量守护者
代码质量专家,执行彻底的代码审查。掌握最佳实践、设计模式和代码异味。确保代码清洁、可维护并遵循团队标准。

**使用场景:** 审查拉取请求、建立代码标准、识别技术债务、指导开发人员或提高代码质量。

### [**compliance-auditor**](compliance-auditor.md) - 法规合规专家
合规专家,确保遵守法规和标准。掌握 GDPR、HIPAA、SOC2 和行业特定要求。以专业知识应对复杂的合规环境。

**使用场景:** 实现法规合规、实施数据隐私、准备审计、记录合规性或理解法规。

### [**debugger**](debugger.md) - 高级调试专家
调试专家,解决最复杂的问题。掌握跨语言和平台的调试工具、技术和方法。在其他人放弃的地方找到根本原因。

**使用场景:** 调试复杂问题、分析内存泄漏、调查竞态条件、分析应用程序或解决间歇性错误。

### [**error-detective**](error-detective.md) - 错误分析和解决专家
错误调查专家,追踪难以捉摸的错误。精通日志分析、错误模式和系统化调试。将神秘错误转化为可执行的解决方案。

**使用场景:** 调查生产错误、分析错误模式、设置错误跟踪、改进错误处理或调试分布式系统。

### [**penetration-tester**](penetration-tester.md) - 道德黑客专家
安全专家,模拟攻击以发现漏洞。掌握 OWASP Top 10、渗透测试工具和利用技术。像攻击者一样思考,像专家一样防御。

**使用场景:** 执行安全评估、测试漏洞、验证安全修复、实施安全测试或准备外部审计。

### [**performance-engineer**](performance-engineer.md) - 性能优化专家
性能专家,使应用程序极快。掌握分析、优化技术和性能测试。消除瓶颈并优化资源使用。

**使用场景:** 优化应用程序性能、进行负载测试、分析瓶颈、改善响应时间或减少资源消耗。

### [**powershell-security-hardening**](powershell-security-hardening.md) - PowerShell 驱动的安全基线专家
以安全为重点的 PowerShell 专家,加固 Windows 服务器、工作站和自动化工具。专注于安全远程处理、受约束端点、凭据卫生、日志记录以及使脚本符合安全基线。

**使用场景:** 加固 Windows 主机、保护 PowerShell 远程处理、锁定脚本和计划任务,或使基础设施自动化符合安全基线和合规要求。

### [**qa-expert**](qa-expert.md) - 测试自动化专家
质量保证大师,设计全面的测试策略。精通测试自动化、框架和方法。通过系统化测试方法确保质量。

**使用场景:** 设置测试自动化、设计测试策略、实施 CI/CD 测试、提高测试覆盖率或建立 QA 流程。

### [**security-auditor**](security-auditor.md) - 安全漏洞专家
安全专家,进行彻底的安全审计。掌握漏洞评估、安全最佳实践和修复策略。保护应用程序免受不断演变的威胁。

**使用场景:** 审计应用程序安全、实施安全最佳实践、修复漏洞、设计安全架构或培训团队安全知识。

### [**test-automator**](test-automator.md) - 测试自动化框架专家
自动化专家,构建健壮的测试框架。精通各种测试工具、模式和策略。创建可维护、可靠的自动化测试套件。

**使用场景:** 构建测试框架、自动化测试用例、将测试集成到 CI/CD、提高测试可靠性或扩展测试自动化。

## 最佳实践

- **左移:**在开发早期发现问题
- **重复自动化:**手动测试无法扩展
- **全程安全:**安全不是事后考虑
- **性能重要:**用户期望快速应用程序
- **包含无障碍:**为所有用户设计
- **持续测试:**质量是持续的
- **监控生产:**从实际使用中学习
- **记录发现:**与团队分享知识
