# 基础设施子代理

基础设施子代理是您的 DevOps 和云计算专家，专门负责构建、部署和维护现代基础设施。这些专家处理从 CI/CD 管道到云架构、从容器编排到数据库管理的所有事务。他们确保您的应用程序可靠运行、高效扩展并在任何环境中无缝部署。

## 何时使用基础设施子代理

在以下情况下使用这些子代理：
- **设计云架构**以实现可扩展性和可靠性
- **实施 CI/CD 管道**以实现自动化部署
- **使用 Kubernetes 和 Docker 编排容器**
- **使用现代工具管理基础设施即代码**
- **优化数据库性能**和管理
- **设置监控和可观测性**系统
- **响应事件**并确保高可用性
- **保护基础设施**并实施最佳实践

## 可用子代理

### [**azure-infra-engineer**](azure-infra-engineer.md) - Azure 云基础设施和自动化专家
精通 Azure 资源设计、虚拟网络、身份集成以及通过 PowerShell、Bicep 和 Az 模块实现的基础设施即代码模式。

**使用场景：** 设计 Azure 环境、安全部署资源、与 M365 集成，或为 Azure 服务和混合身份创建自动化脚本。

### [**cloud-architect**](cloud-architect.md) - AWS/GCP/Azure 专家
多云专家，设计可扩展、成本优化的云解决方案。精通云原生架构、无服务器模式和云迁移策略。确保主要云提供商的最佳资源利用。

**使用场景：** 设计云架构、迁移到云、优化云成本、实施多云策略或选择云服务。

### [**database-administrator**](database-administrator.md) - 数据库管理专家
数据库专家，大规模管理关系型和 NoSQL 数据库。精通性能调优、复制、备份策略和高可用性。确保数据完整性和最佳数据库性能。

**使用场景：** 设置数据库、优化查询性能、实施备份策略、设计数据库架构或排查数据库问题。

### [**deployment-engineer**](deployment-engineer.md) - 部署自动化专家
部署专家，跨环境自动化应用程序发布。精通蓝绿部署、金丝雀发布和回滚策略。确保零停机时间的自信部署。

**使用场景：** 设置部署管道、实施发布策略、自动化部署、管理环境或确保部署可靠性。

### [**devops-engineer**](devops-engineer.md) - CI/CD 和自动化专家
DevOps 实践者，连接开发和运营。精通 CI/CD 管道、自动化工具和 DevOps 文化。加速交付的同时保持稳定性和安全性。

**使用场景：** 构建 CI/CD 管道、自动化工作流、实施 DevOps 实践、设置开发环境或提高部署速度。

### [**devops-incident-responder**](devops-incident-responder.md) - DevOps 事件管理
DevOps 环境的事件响应专家。精通故障排除、根因分析和事件管理。通过系统化方法最小化停机时间并防止未来事件。

**使用场景：** 响应生产事件、设置事件管理流程、执行根因分析或实施事件预防措施。

### [**incident-responder**](incident-responder.md) - 系统事件响应专家
处理系统中断和紧急情况的关键事件专家。精通快速诊断、恢复程序和事后分析。快速恢复服务的同时从失败中学习。

**使用场景：** 管理关键事件、制定事件响应计划、进行事后分析或培训事件响应团队。

### [**kubernetes-specialist**](kubernetes-specialist.md) - 容器编排大师
Kubernetes 专家，大规模管理容器化应用程序。精通集群设计、工作负载优化和 Kubernetes 生态系统工具。确保生产环境中可靠的容器编排。

**使用场景：** 部署到 Kubernetes、设计集群架构、优化工作负载、实施服务网格或排查 Kubernetes 问题。

### [**network-engineer**](network-engineer.md) - 网络基础设施专家
网络架构专家，设计安全、高性能的网络。精通 SDN、负载均衡和网络安全。确保可靠的连接性和最佳网络性能。

**使用场景：** 设计网络架构、实施负载均衡器、设置 VPN、优化网络性能或排查连接问题。

### [**platform-engineer**](platform-engineer.md) - 平台架构专家
平台专家，构建内部开发者平台。创建自助式基础设施、黄金路径和平台抽象。在保持治理的同时赋能开发者。

**使用场景：** 构建内部平台、创建开发者门户、实施平台工程、标准化基础设施或提高开发者生产力。

### [**security-engineer**](security-engineer.md) - 基础设施安全专家
保护基础设施和应用程序的安全专家。精通安全加固、合规性和威胁预防。在所有层面实施纵深防御策略。

**使用场景：** 保护基础设施、实施安全策略、实现合规性、执行安全审计或响应安全事件。

### [**sre-engineer**](sre-engineer.md) - 站点可靠性工程专家
通过工程确保系统可靠性的 SRE 实践者。精通 SLI/SLO、错误预算和混沌工程。平衡功能速度与系统稳定性。

**使用场景：** 实施 SRE 实践、定义 SLO、设置监控、执行混沌工程或提高系统可靠性。

### [**terraform-engineer**](terraform-engineer.md) - 基础设施即代码专家
使用 Terraform 实现基础设施自动化的 IaC 专家。精通模块设计、状态管理和多环境部署。确保基础设施的一致性和可重复性。

**使用场景：** 编写 Terraform 代码、设计 IaC 架构、管理 Terraform 状态、创建可重用模块或自动化基础设施配置。

### [**terragrunt-expert**](terragrunt-expert.md) - Terragrunt 编排和 DRY IaC 专家
高级 Terragrunt 专家，大规模编排 OpenTofu/Terraform 基础设施。精通堆栈架构、单元组合、依赖管理和 DRY 配置模式。确保企业级基础设施自动化，专注于代码重用和可维护性。

**使用场景：** 使用 Terragrunt 编排 Terraform 模块、跨环境实施 DRY 配置、管理复杂依赖图、设计多账户/多区域基础设施，或从单体 Terraform 迁移到模块化 Terragrunt 堆栈。

### [**windows-infra-admin**](windows-infra-admin.md) - Windows 基础设施和 Active Directory 自动化专家
在使用 PowerShell 自动化 AD、DNS、DHCP、GPO、服务器配置和域服务方面具有深厚专业知识。专注于安全变更工作流、幂等操作和企业级运营模式。

**使用场景：** 管理域基础设施、修改 AD 对象、更新 DNS/DHCP 记录、自动化 GPO 任务，或在企业环境中执行服务器级自动化。

## 入门指南

1. **评估您的基础设施需求**和当前挑战
2. **根据您的需求选择合适的专家**
3. **提供关于您的环境和约束的上下文**
4. **如果适用，共享现有配置**
5. **遵循专家的最佳实践建议**

## 最佳实践

- **从架构开始：** 先设计后实施
- **自动化一切：** 手动流程无法扩展
- **安全优先：** 将安全性构建到每一层
- **主动监控：** 可观测性预防事件
- **彻底记录：** 未来的您会感谢您
- **测试基础设施：** 基础设施代码也需要测试
- **规划失败：** 为弹性而设计
- **持续迭代：** 基础设施随需求演进
