# 业务与产品子代理

业务与产品子代理在技术和业务价值之间架起桥梁。这些专家既了解技术实现又懂得业务策略，帮助团队构建用户喜爱且企业蓬勃发展的产品。从产品策略到客户成功，从业务分析到技术写作，他们确保技术服务于真实的业务需求并交付可测量的价值。

## 何时使用业务与产品子代理

在以下情况下使用这些子代理：
- **定义产品策略**和路线图
- **分析业务需求**并转换为技术规格
- **进行用户研究**以验证想法
- **创建内容**推动参与
- **管理客户关系**和成功
- **确保法律合规**的技术决策
- **有效管理项目**使用敏捷方法
- **架起技术与业务**沟通桥梁

## 可用的子代理

### [**business-analyst**](business-analyst.md) - 需求专家
业务分析专家将业务需求转化为技术要求。精通利益相关者沟通、流程分析和解决方案设计。确保技术解决真实的业务问题。

**使用场景：** 收集需求、分析业务流程、定义规格、创建用户故事或架起业务与技术沟通的桥梁。

### [**content-marketer**](content-marketer.md) - 内容营销专家
内容专家创建引人注目的技术和营销内容。精通 SEO、内容策略和受众参与。通过战略性内容创作推动增长。

**使用场景：** 创建博客文章、制定内容策略、撰写营销文案、优化 SEO 或构建内容日历。

### [**customer-success-manager**](customer-success-manager.md) - 客户成功专家
客户成功专家确保用户实现其目标。擅长入职、留存和客户倡导。通过主动支持将用户转变为拥护者。

**使用场景：** 设计入职流程、提高用户留存、收集客户反馈、构建成功指标或创建客户计划。

### [**legal-advisor**](legal-advisor.md) - 法律与合规专家
法律专家引导技术法律和合规。精通隐私法规、知识产权和合同谈判。在实现创新的同时保护企业。

**使用场景：** 审查服务条款、确保数据隐私合规、理解许可、管理知识产权或评估法律风险。

### [**product-manager**](product-manager.md) - 产品策略专家
产品愿景家定义构建什么以及为什么。擅长市场分析、用户需求和产品策略。从构思到市场领导地位推动产品成功。

**使用场景：** 定义产品愿景、优先排序功能、进行市场研究、创建路线图或做出产品决策。

### [**project-manager**](project-manager.md) - 项目管理专家
项目管理专家确保成功交付。精通敏捷方法、资源规划和利益相关者管理。保持项目按时、按预算和按目标推进。

**使用场景：** 规划项目、管理时间线、协调团队、跟踪进度或实施项目方法。

### [**sales-engineer**](sales-engineer.md) - 技术销售专家
销售工程专家在技术复杂性和客户需求之间架起桥梁。擅长演示、POC 和技术异议。帮助客户理解和采用技术解决方案。

**使用场景：** 创建技术演示、处理销售异议、设计 POC、支持销售团队或解释技术价值。

### [**scrum-master**](scrum-master.md) - 敏捷方法专家
敏捷促进者确保团队有效工作。精通 Scrum 框架、团队动态和持续改进。消除障碍并培养高绩效团队。

**使用场景：** 实施 Scrum、促进仪式、改进团队流程、消除障碍或指导敏捷实践。

### [**technical-writer**](technical-writer.md) - 技术文档专家
文档专家使复杂的技术概念易于理解。精通各种文档类型、工具和以用户为中心的写作。创建用户真正阅读的文档。

**使用场景：** 编写用户指南、创建 API 文档、开发教程、改进文档或构建知识库。

### [**ux-researcher**](ux-researcher.md) - 用户研究专家
用户研究专家揭示用户需求和行为。擅长研究方法、可用性测试和洞察综合。确保产品建立在真实的用户理解之上。

**使用场景：** 进行用户访谈、运行可用性测试、分析用户行为、创建人物角色或验证产品决策。

## 常见业务模式

**产品开发：**
- **product-manager** 负责愿景
- **ux-researcher** 负责用户洞察
- **business-analyst** 负责需求
- **project-manager** 负责执行

**市场推广：**
- **content-marketer** 负责内容
- **sales-engineer** 负责演示
- **technical-writer** 负责文档
- **customer-success-manager** 负责留存

**敏捷团队：**
- **scrum-master** 负责流程
- **product-manager** 负责优先级
- **business-analyst** 负责故事
- **project-manager** 负责跟踪

**客户聚焦：**
- **ux-researcher** 负责理解
- **customer-success-manager** 负责满意度
- **technical-writer** 负责自助服务
- **sales-engineer** 负责采用

## 入门指南

1. **明确识别业务目标**
2. **选择与目标一致的专家**
3. **提供业务上下文**和约束
4. **促进专家之间的协作**
5. **持续测量业务影响**

## 最佳实践

- **以用户为中心的方法：** 始终考虑最终用户
- **数据驱动决策：** 测量和验证
- **清晰沟通：** 架起技术与业务的桥梁
- **迭代改进：** 小步骤，大影响
- **利益相关者对齐：** 让每个人都知情
- **文档很重要：** 知识应该可访问
- **法律合规：** 尽早考虑法规
- **业务价值聚焦：** 技术服务于业务目标
