# 开发者体验子代理

开发者体验子代理是您的生产力倍增器,专注于让开发更快、更简单、更愉悦。这些专家处理从代码重构到文档编写、从构建优化到 Git 工作流程的所有事务。他们消除开发过程中的摩擦,自动化重复性任务,并通过更好的工具和实践帮助团队更高效地工作。

## 何时使用开发者体验子代理

在以下情况下使用这些子代理:
- **重构遗留代码**以提高可维护性
- **优化构建系统**以加快开发速度
- **创建开发者工具**和 CLI 应用程序
- **编写技术文档**让开发者喜爱
- **管理依赖项**和包更新
- **精简 Git 工作流程**和分支策略
- **现代化代码库**采用最新实践
- **提高开发者生产力**跨团队

## 可用的子代理

### [**build-engineer**](build-engineer.md) - 构建系统专家
构建优化专家,让编译和打包变得极速。精通各种构建工具、优化技术和缓存策略。将构建时间从几分钟缩短到几秒钟。

**使用场景:** 优化构建时间、配置构建工具、实现构建缓存、设置 monorepo 构建,或排查构建问题。

### [**cli-developer**](cli-developer.md) - 命令行工具和自动化专家
高级 CLI 工程师,为开发者和运维人员构建直观高效的命令行工具。专长于参数解析、交互式提示、终端 UX 和跨平台兼容性,重点关注易于脚本化的接口以及与现有工作流程的平滑集成。

**使用场景:** 设计或重构内部工具、DevOps/运维 CLI、PowerShell/Bash 包装器,或任何需要易于发现、符合人体工程学且易于在流水线中自动化的命令行体验。

### [**dependency-manager**](dependency-manager.md) - 包和依赖项专家
依赖项专家,管理复杂的包生态系统。精通版本解析、安全更新和依赖项优化。在不破坏系统的前提下保持依赖项安全和最新。

**使用场景:** 管理依赖项、解决版本冲突、实施安全更新、优化包大小,或设置依赖项自动化。

### [**documentation-engineer**](documentation-engineer.md) - 技术文档专家
文档专家,创建清晰全面的技术文档。精通 API 文档、教程和开发者指南。通过优秀的文档使复杂系统易于理解。

**使用场景:** 编写 API 文档、创建开发者指南、构建文档站点、改进现有文档,或设置文档工作流程。

### [**dx-optimizer**](dx-optimizer.md) - 开发者体验优化专家
DX 专家,识别并消除开发者摩擦。分析工作流程、工具和流程以提高生产力。让开发感觉轻松愉快。

**使用场景:** 改进开发者工作流程、分析生产力瓶颈、选择开发者工具、优化开发环境,或衡量开发者体验。

### [**git-workflow-manager**](git-workflow-manager.md) - Git 工作流程和分支专家
Git 专家,设计高效的版本控制工作流程。精通分支策略、合并冲突解决和 Git 自动化。通过 Git 最佳实践确保顺畅协作。

**使用场景:** 设计 Git 工作流程、实施分支策略、解决复杂合并、自动化 Git 流程,或培训团队使用 Git。

### [**legacy-modernizer**](legacy-modernizer.md) - 遗留代码现代化专家
现代化专家,为旧代码库注入新生命。精通渐进式重构、依赖项更新和架构改进。在不破坏功能的前提下改造遗留代码。

**使用场景:** 现代化遗留应用、规划重构策略、更新旧框架、迁移到新技术,或提高代码可维护性。

### [**mcp-developer**](mcp-developer.md) - 模型上下文协议专家
MCP 专家,构建连接 AI 系统与外部工具和数据源的服务器和客户端。精通协议规范、SDK 实现和生产就绪集成。在 AI 和外部服务之间创建无缝桥梁。

**使用场景:** 构建 MCP 服务器、创建 AI 工具集成、实现模型上下文协议客户端、连接 AI 系统到外部 API,或开发具有外部数据源的 AI 驱动应用程序。

### [**powershell-module-architect**](powershell-module-architect.md) - PowerShell 模块和配置文件架构专家
PowerShell 架构专家,将临时脚本转化为清晰可重用的模块和快速加载的配置文件。专注于清晰的公共/私有函数边界、健壮的参数设计、DRY 辅助库,以及 Windows PowerShell 5.1 和 PowerShell 7+ 之间的跨版本兼容性。

**使用场景:** 构建或重构 PowerShell 模块、精简缓慢的配置文件、设计函数/参数约定,或为系统管理员和服务台组织共享基础设施工具。

### [**powershell-ui-architect**](powershell-ui-architect.md) - PowerShell GUI 和 TUI 专家
PowerShell 工具的 UI 和 UX 架构师,在自动化模块之上设计 WinForms、WPF、Metro 风格仪表板(MahApps.Metro/Elysium)和终端 UI。专注于在可重用的 PowerShell/.NET 逻辑之上分层清晰的界面,而不牺牲可维护性。

**使用场景:** 您需要为 PowerShell 工具构建图形界面或终端 UI,想在 WinForms/WPF/TUI/Metro 方法之间选择,或需要帮助围绕现有 PowerShell 模块和脚本构建 XAML 和事件处理程序。

### [**refactoring-specialist**](refactoring-specialist.md) - 代码重构专家
重构大师,在不改变行为的前提下改进代码结构。精通设计模式、代码异味和安全重构技术。使代码更清晰、更可维护。

**使用场景:** 重构复杂代码、消除代码异味、实现设计模式、改进代码结构,或为新功能准备代码。

### [**slack-expert**](slack-expert.md) - Slack 平台和 @slack/bolt 专家
精英 Slack 平台专家,在 @slack/bolt、Slack Web API、Events API、Block Kit UI 和 OAuth 流程方面拥有深厚专业知识。使用速率限制、安全性和现代功能的最佳实践构建强大的 Slack 集成。

**使用场景:** 构建 Slack 机器人、实现斜杠命令、创建 Block Kit 界面、审查 Slack 代码、设置 OAuth 流程,或与 Slack 的 Events API 集成。

### [**tooling-engineer**](tooling-engineer.md) - 开发者工具专家
工具专家,构建和集成开发者工具。精通 IDE 配置、代码检查器、格式化器和自定义工具。创建提升生产力的开发环境。

**使用场景:** 设置开发工具、创建自定义工具、配置 IDE、实现代码质量工具,或构建开发者平台。

## 最佳实践

- **自动化重复性任务:** 节省的时间会累积
- **记录一切:** 未来的开发者会感谢您
- **渐进式改进:** 小改变会累积
- **衡量影响:** 跟踪生产力提升
- **工具标准化:** 一致性减少摩擦
- **开发者反馈:** 倾听您的用户
- **持续改进:** DX 永远不会"完成"
- **分享知识:** 传播最佳实践
