# 核心开发子代理

核心开发子代理是您从零开始构建现代应用的必备工具包。这些专业化代理覆盖了整个开发领域——从后端服务到前端界面，从移动应用到桌面应用，从简单 API 到复杂的分布式系统。

## 🎯 何时使用核心开发子代理

在以下情况下使用这些子代理：
- **从零构建新应用**，并采用合理的架构设计
- **实现复杂功能**，需要深厚的技术专长
- **设计可扩展的系统**，能够随业务需求增长
- **创建精美的 UI**，提供卓越的用户体验
- **开发实时功能**，用于交互式应用
- **现代化改造遗留系统**，采用当前最佳实践
- **优化性能**，涵盖整个技术栈

## 📋 可用子代理

### [**api-designer**](api-designer.md) - REST 和 GraphQL API 架构师
设计美观、直观且可扩展 API 的架构师。精通 RESTful 原则、GraphQL 模式、API 版本管理和文档编写。确保您的 API 对开发者友好且面向未来。

**使用场景：** 设计新 API、重构现有端点、实施 API 标准或创建全面的 API 文档。

### [**backend-developer**](backend-developer.md) - 构建可扩展 API 的服务端专家
构建稳健服务端应用、RESTful API 和微服务的首选专家。擅长数据库设计、认证系统和性能优化。适合使用 Node.js、Python、Java 或其他后端技术构建应用的核心骨架。

**使用场景：** 构建 API、设计数据库、实现认证、处理业务逻辑或优化服务器性能。

### [**electron-pro**](electron-pro.md) - 桌面应用专家
专注于使用 Web 技术构建跨平台桌面应用的专家。精通 Electron 框架，能创建具备原生能力的可安装桌面应用。处理自动更新、系统集成和桌面特有功能。

**使用场景：** 创建桌面应用、将 Web 应用移植到桌面、实现系统托盘功能或构建支持离线的桌面工具。

### [**frontend-developer**](frontend-developer.md) - React、Vue 和 Angular UI/UX 专家
现代 Web 界面大师，创建响应式、无障碍且高性能的用户体验。精通组件架构、状态管理和现代 CSS。将设计稿转化为像素级精确的交互式应用。

**使用场景：** 创建 Web 界面、实现复杂 UI 组件、优化前端性能或确保无障碍合规性。

### [**fullstack-developer**](fullstack-developer.md) - 端到端功能开发
全栈多面手，能无缝地跨越整个技术栈工作。从数据库到 UI 构建完整功能，确保前后端之间的顺畅集成。适合快速原型开发和完整功能实现。

**使用场景：** 构建完整功能、原型设计应用、中小型项目开发，或需要跨技术栈统一开发时使用。

### [**graphql-architect**](graphql-architect.md) - GraphQL 模式与联邦架构专家
专注于 GraphQL 生态系统，从模式设计到联邦策略。精通解析器优化、订阅模式和 GraphQL 最佳实践。适合构建灵活、高效的数据层。

**使用场景：** 实现 GraphQL API、设计模式、优化解析器、配置联邦架构或从 REST 迁移到 GraphQL。

### [**microservices-architect**](microservices-architect.md) - 分布式系统设计师
设计和实现微服务架构的专家。处理服务拆分、服务间通信、分布式事务和编排。确保您的系统具备水平扩展和弹性恢复能力。

**使用场景：** 将单体应用拆分为微服务、设计分布式系统、实施服务网格或解决分布式系统挑战。

### [**mobile-developer**](mobile-developer.md) - 跨平台移动开发专家
精通为 iOS 和 Android 创建原生和跨平台移动应用。熟练使用 React Native、Flutter 和原生开发。专注于移动端特有挑战，如离线功能、推送通知和应用商店优化。

**使用场景：** 构建移动应用、实现移动端特有功能、优化移动端性能或准备应用商店上架。

### [**ui-designer**](ui-designer.md) - 视觉设计与交互专家
视觉设计大师，创建美观、直观且无障碍的用户界面。精通设计系统、排版、色彩理论和交互模式。将创意转化为精致的设计，在美学与功能之间取得平衡，同时保持品牌一致性。

**使用场景：** 创建视觉设计、构建设计系统、定义交互模式、建立品牌标识或准备设计交付给开发团队。

### [**websocket-engineer**](websocket-engineer.md) - 实时通信专家
实时双向通信大师。实现 WebSocket 服务器、大规模管理连接，处理聊天、通知和实时更新等实时功能。精通 Socket.io 和原生 WebSocket 实现。

**使用场景：** 构建聊天应用、实现实时通知、创建协作功能或开发实时更新仪表板。

### [**wordpress-master**](wordpress-master.md) - WordPress 开发与优化专家
WordPress 生态系统专家，能构建从简单博客到企业级平台的各类项目。精通主题开发、插件架构、Gutenberg 区块和性能优化。在传统 PHP 开发和现代区块化解决方案方面均有丰富经验。

**使用场景：** 构建 WordPress 站点、开发自定义主题、创建插件、实现 WooCommerce 解决方案或优化 WordPress 性能。

## 💡 常见组合

**全栈 Web 应用：**
- 先用 **api-designer** 设计 API 结构
- 再用 **backend-developer** 实现服务端
- 最后用 **frontend-developer** 开发 UI

**企业级系统：**
- 先用 **microservices-architect** 设计系统架构
- 再用 **graphql-architect** 构建数据层
- 最后用 **backend-developer** 实现各服务

**实时应用：**
- 先用 **websocket-engineer** 搭建实时基础设施
- 再用 **backend-developer** 处理业务逻辑
- 最后用 **frontend-developer** 构建交互式 UI

**设计驱动开发：**
- 先用 **ui-designer** 进行视觉设计和原型制作
- 再用 **frontend-developer** 实现开发
- 最后用 **accessibility-tester** 进行合规性验证

**WordPress 项目：**
- 先用 **wordpress-master** 进行架构和配置
- 再用 **php-pro** 进行自定义 PHP 开发
- 最后用 **frontend-developer** 开发自定义 JavaScript

## 🎬 开始使用

1. **选择合适的子代理**，根据您的具体需求
2. **提供清晰的上下文**，说明您的项目需求
3. **指定技术栈**偏好（如有）
4. **描述约束条件**（性能、可扩展性、时间表）
5. **让子代理引导您**完成最佳实践和实现

每个子代理都具备：
- 所在领域的深厚专业知识
- 了解当前最佳实践
- 能够与现有代码库配合工作
- 注重代码整洁和可维护性
- 理解生产环境需求

## 📚 最佳实践

- **从架构开始：** 在实现之前先使用架构师（API、GraphQL、微服务）
- **频繁迭代：** 以短周期与子代理协作以获得更好的结果
- **组合专长：** 复杂项目使用多个子代理
- **遵循规范：** 每个子代理都了解其领域的最佳实践
- **面向生产：** 所有子代理都会考虑可扩展性、安全性和可维护性
