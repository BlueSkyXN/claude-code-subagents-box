# 专业领域子代理

专业领域子代理是您在特定技术垂直领域和行业的专家。这些专家带来了对特定领域挑战、法规和最佳实践的深入了解。从区块链和物联网到金融科技和游戏，他们理解各自专业领域的独特要求和模式，帮助您构建在这些复杂领域表现出色的应用程序。

## 何时使用专业领域子代理

在以下情况下使用这些子代理：
- **构建区块链应用**和智能合约
- **开发物联网解决方案**用于互联设备
- **创建支付系统**与各种提供商
- **构建游戏应用**具有实时功能
- **实施金融科技解决方案**具有合规性
- **开发嵌入式系统**具有硬件约束
- **创建移动应用**具有原生功能
- **设计金融算法**用于交易系统

## 可用子代理

### [**api-documenter**](api-documenter.md) - API文档专家
API文档专家创建开发者友好的API文档。精通OpenAPI/Swagger、交互式文档和API最佳实践。使API易于发现和集成。

**使用时机：** 记录REST API、创建API规范、构建开发者门户、生成客户端SDK或改进API可发现性。

### [**blockchain-developer**](blockchain-developer.md) - Web3和加密专家
区块链专家构建去中心化应用和智能合约。精通Ethereum、Solidity和Web3技术。创建安全、高效的区块链解决方案。

**使用时机：** 构建dApp、编写智能合约、实施DeFi协议、创建NFT平台或集成区块链功能。

### [**embedded-systems**](embedded-systems.md) - 嵌入式和实时系统专家
嵌入式系统专家在受限环境中工作。精通微控制器、RTOS和硬件接口。为资源有限的设备构建高效软件。

**使用时机：** 编程微控制器、开发固件、实施实时系统、优化内存/功耗或与硬件接口。

### [**fintech-engineer**](fintech-engineer.md) - 金融科技专家
金融科技专家构建安全、合规的金融应用。精通支付处理、监管要求和金融API。导航复杂的金融技术世界。

**使用时机：** 构建支付系统、实施银行功能、确保金融合规、集成金融API或开发交易平台。

### [**game-developer**](game-developer.md) - 游戏开发专家
游戏专家创建引人入胜的互动体验。精通游戏引擎、实时网络和性能优化。构建吸引玩家的游戏。

**使用时机：** 开发游戏、实施游戏机制、优化游戏性能、构建多人功能或创建游戏工具。

### [**iot-engineer**](iot-engineer.md) - 物联网系统开发者
物联网专家将物理设备连接到云。精通设备协议、边缘计算和物联网平台。为物联网创建可扩展的解决方案。

**使用时机：** 构建物联网应用、实施设备通信、管理物联网车队、处理传感器数据或设计物联网架构。

### [**mobile-app-developer**](mobile-app-developer.md) - 移动应用专家
移动专家创建原生和跨平台应用程序。精通iOS/Android开发、移动UI/UX和应用商店部署。构建用户喜爱的设备应用。

**使用时机：** 创建移动应用、实施原生功能、优化移动性能、处理离线功能或部署到应用商店。

### [**payment-integration**](payment-integration.md) - 支付系统专家
支付专家集成各种支付提供商和方法。精通PCI合规、支付安全和交易处理。使支付无缝且安全。

**使用时机：** 集成支付网关、实施订阅、处理PCI合规、处理交易或构建结账流程。

### [**quant-analyst**](quant-analyst.md) - 量化分析专家
量化专家开发金融算法和模型。精通统计分析、风险建模和算法交易。将市场数据转化为盈利策略。

**使用时机：** 构建交易算法、开发风险模型、分析金融数据、实施量化策略或回测系统。

### [**risk-manager**](risk-manager.md) - 风险评估和管理专家
风险管理专家识别和缓解各种风险。精通风险建模、合规和缓解策略。保护系统和业务免受潜在威胁。

**使用时机：** 评估技术风险、实施风险控制、构建风险模型、确保合规或开发风险管理系统。

### [**seo-specialist**](seo-specialist.md) - 搜索引擎优化专家
SEO专家通过搜索优化推动自然流量。精通技术SEO、内容策略和链接建设。通过数据驱动策略提高搜索排名和可见性。

**使用时机：** 优化搜索引擎、实施结构化数据、提高网站速度、构建内容策略或分析搜索性能。

3. **考虑监管合规性**（如适用）
4. **提前规划特定领域的挑战**
5. **在整个开发过程中利用领域专业知识**

## 最佳实践

- **领域知识很重要：** 深入了解领域
- **合规至关重要：** 许多领域都有法规
- **安全优先：** 专业领域通常处理敏感数据
- **性能要求：** 每个领域都有独特的需求
- **用户期望：** 领域用户有特定的工作流程
- **行业标准：** 遵循既定模式
- **保持更新：** 专业领域快速发展
- **彻底测试：** 特定领域的边缘情况很重要
